# DPLL-based Boolean Satisfiability Solver

课程属性：数据结构与算法课程项目 1  
原始材料：`raw/23339050_proj1.cpp`, `raw/23339050_proj1.pdf`

## 项目目标

本项目针对由若干三文字子句构成的 3-CNF 布尔可满足性问题，使用 DPLL 算法寻找一组变量赋值，使所有子句同时为真。若不存在满足赋值，程序输出错误提示。

## 数据结构设计

- `Clause`: 使用 `vector<int>` 保存一个子句中的文字。
- `vector<Clause>`: 保存整个 CNF 公式。
- `vector<int> assignment`: 保存变量状态，其中 `1` 表示真，`0` 表示未赋值，`-1` 表示假。

## 算法流程

1. 读取变量数 `n` 与子句数 `m`。
2. 读取每个子句中的三个文字，并进行输入合法性检查。
3. 使用当前赋值简化子句集合：
   - 已满足的子句从公式中删除。
   - 已为假的文字从子句中删除。
   - 若出现空子句，说明当前分支矛盾。
4. 若出现单子句，直接给对应变量赋值并继续简化。
5. 若没有单子句，选择出现频率最高的未赋值变量，分别尝试真与假。
6. 若所有子句被满足，输出一组可行赋值。

```bash
./dpll_solver
```


## 输入输出示例

输入：

```text
3
2
1 -2 3
-1 2 3
```

输出可能为：

```text
1 1 1
```

说明：输出为变量 `x1, x2, x3` 的一组满足赋值。由于 DPLL 分支选择不同，满足赋值不唯一。

## 复杂度分析

- 最坏情况下，DPLL 仍可能枚举指数级赋值，时间复杂度为 `O(2^n * m)` 量级。
- 子句简化会遍历当前子句集合，单次简化约为 `O(m)`。
- 空间主要来自子句集合、赋值向量和递归中的临时赋值拷贝。
